1. 프로젝트 개요

1.1 프로젝트 목표

유아 교육을 위한 인터랙티브 동화책 웹 플랫폼 구축

  • 직관적이고 아이들이 쉽게 사용할 수 있는 UI/UX
  • 이미지와 텍스트가 결합된 동화책 뷰어
  • TTS(Text-to-Speech) 음성 읽기 기능
  • 관리자 페이지를 통한 쉬운 콘텐츠 관리
  • 모바일 반응형 디자인

1.2 핵심 가치

  • 접근성: 웹 기반으로 어디서나 접근 가능
  • 교육성: TTS를 통한 듣기 학습 지원
  • 확장성: 관리자 페이지로 쉽게 콘텐츠 추가
  • 안정성: JSON 기반 데이터 관리로 간단하고 안정적

1.3 타겟 사용자

  • 주 사용자: 3-7세 유아 및 초등 저학년
  • 보조 사용자: 학부모, 교육 기관
  • 관리자: 콘텐츠 제작자, 교육 콘텐츠 관리자

2. 기술 스택 및 아키텍처

2.1 기술 스택

구분 기술 비고
Backend Node.js + Express RESTful API 서버
Frontend Vanilla JavaScript + HTML5 + CSS3 프레임워크 없이 경량화
스타일 Tailwind CSS (CDN) + Custom CSS 빠른 개발과 커스터마이징
데이터 JSON 파일 데이터베이스 불필요
이미지 외부 URL (ImgBB, CDN) 서버 저장공간 절약
TTS Web Speech API 브라우저 내장 기능
배포 PM2 (개발) / Vercel (프로덕션) 간편한 배포
버전 관리 Git + GitHub 코드 관리 및 협업

3. 핵심 기능 (구현 완료)

3.1 사용자 페이지

3.1.1 메인 페이지 (index.html)

✅ 구현 완료
  • ✓ 동화책 목록 카드 형태로 표시
  • ✓ 표지 이미지, 제목, 작가, 페이지 수, 태그 표시
  • ✓ 연령대별 필터링 (전체/3-5세/5-7세/4-6세)
  • ✓ 태그별 필터링
  • ✓ 검색 기능 (제목, 작가 검색)
  • ✓ 반응형 그리드 레이아웃
  • ✓ 카드 클릭 시 뷰어로 이동

3.1.2 동화책 뷰어 (reader.html)

✅ 구현 완료
  • ✓ 이미지 + 텍스트 동시 표시
  • ✓ 이전/다음 페이지 네비게이션
  • ✓ 페이지 번호 표시 (현재/전체)
  • ✓ 키보드 단축키 지원 (← → 스페이스바)
  • ✓ TTS 음성 읽기 기능
    • 재생/일시정지/정지 버튼
    • 속도 조절 (느림/보통/빠름)
    • 자동 페이지 넘김
    • 한국어 음성 지원
    • 모바일 브라우저 호환성 개선
  • ✓ TTS 컨트롤 토글 기능 (🎵 버튼)
    • 모바일에서 화면 가림 방지
    • 클릭으로 컨트롤 접기/펼치기
  • ✓ 읽기 진행률 저장 (LocalStorage)
  • ✓ 이미지 로딩 오류 처리 (SVG placeholder)
  • ✓ 모바일 반응형 디자인
  • ✓ Eruda 모바일 디버깅 콘솔

3.2 관리자 페이지

3.2.1 동화책 관리 (admin.html)

✅ 구현 완료
  • ✓ 동화책 목록 표시 (카드 형태)
  • ✓ 공개/비공개 상태 표시
  • ✓ 편집/삭제 버튼
  • ✓ 새 동화책 만들기 버튼

3.2.2 동화책 생성/편집 (3단계 프로세스)

Step 1: 기본 정보 입력
  • ✓ 제목, 작가, 표지 이미지 URL
  • ✓ 연령대 선택
  • ✓ 태그 입력 (쉼표로 구분)
  • ✓ 공개 상태 설정
Step 2: 통합 페이지 편집 (핵심 기능)
  • ✓ 기존 데이터 자동 로드
    • Step 2 진입 시 자동으로 기존 페이지 데이터 표시
    • 이미지 URL과 텍스트가 각 입력란에 자동 채워짐
  • ✓ 이미지 URL + 텍스트 동시 조회/수정
    • 왼쪽: 이미지 URL (한 줄에 하나씩)
    • 오른쪽: 텍스트 (--- 구분자 사용)
    • 나란히 배치되어 동시 확인 가능
  • ✓ 실시간 카운트 및 검증
    • 이미지 개수와 텍스트 개수 실시간 표시
    • 개수 불일치 시 경고 메시지
    • 개수 일치 시 확인 메시지
  • ✓ 페이지별 미리보기
    • 👁️ 미리보기 버튼 클릭
    • 모든 페이지를 소형 카드로 표시
    • 이미지 URL 유효성 자동 검사
    • 로드 실패 시 빨간색 "로드 실패" 표시
  • ✓ 편리한 버튼
    • 🔄 기존 데이터 불러오기
    • 👁️ 미리보기
    • 💾 저장하기
Step 3: 완료 화면
  • ✓ 저장 완료 메시지
  • ✓ 📖 동화책 보기 버튼 (즉시 뷰어로 이동)
  • ✓ 목록으로 버튼

4. API 엔드포인트

4.1 공개 API

Method Endpoint 설명
GET /api/storybooks 전체 동화책 목록 조회
GET /api/storybooks/:id 특정 동화책 상세 조회

4.2 관리자 API

Method Endpoint 설명
POST /api/admin/storybooks 새 동화책 생성
PUT /api/admin/storybooks/:id 동화책 정보 수정
DELETE /api/admin/storybooks/:id 동화책 삭제
POST /api/admin/storybooks/:id/pages/bulk-images 다중 이미지 URL 추가/교체
POST /api/admin/storybooks/:id/pages/bulk-texts 텍스트 일괄 입력
